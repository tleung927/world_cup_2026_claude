# 足球比赛预测 — 多智能体系统

[English](README.md) · **简体中文**

一套基于 [Claude Code](https://claude.com/claude-code) 的足球比赛预测多智能体流水线。多个「顾问型」子智能体各自负责一个数据领域；Claude Code 主会话充当确定性的**编排器（orchestrator）**；再由「偏差审查（Bias Auditor）」与「CEO 否决（CEO/Veto）」两道关卡把关最终结论。

它刻意做成交易台（trading desk）的形态：专业分析师把信息喂给控制者，任何结论出手前先过一道风险/偏差检查，最后由资深角色行使否决权。大语言模型智能体**仅提供建议** —— 流水线的顺序与最终汇总是**确定性的**，由 `CLAUDE.md` 定义。

> **免责声明。** 本项目仅供学习/实验用途。预测由 AI 生成，可能出错，**不构成任何投注建议**。请勿据此下注。

---

## 为什么这样设计

这套方法论是在一长串真实的 2026 世界杯预测中「用血泪」磨出来的。最大的一条教训是：**方向（谁赢、大胜还是小胜、是否双方都进球）是真正可预测的；而精确比分基本靠运气。** 因此各智能体被调校为：对**方向**充满信心，对**精度的极限**保持诚实。请看 `knowledge/lessons-learned.md` —— 它是整个系统的核心，也是「偏差审查」存在的理由。

## 智能体 —— 6 个子智能体 + 1 个编排器

流水线由 **6 个项目级子智能体**构成。所谓的「第 7 个」是编排器，它**不是**一个子智能体文件 —— 而是 Claude Code 主会话本身，由 `CLAUDE.md` 管辖。

| # | 智能体 | 职责 | 关卡 |
|---|-------|------|-------|
| 1 | **match-type-classifier**（类型分类器） | 判断这是「哪一类比赛」，从而套用正确的逻辑。因为下游一切都依赖它，所以最先运行。 | 关卡 1 |
| 2 | **match-analyst**（比赛分析 / 「教练」） | 球队实力差、战术对位、赛事利害/动机、主场优势。产出核心研判。 | — |
| 3 | **weather-analyst**（天气分析） | 气温、降雨、**顶棚开/闭**、海拔、延期风险。环境会放大实力差。 | — |
| 4 | **squad-analyst**（阵容分析） | 确认首发、伤停、状态、关键射手、**门将情况**、轮换风险。 | — |
| 5 | **bias-auditor**（偏差审查） | 在结论出手*之前*，把草案对照「已记录的常犯错误清单」逐条核对。纯机械式检查表。 | 关卡 2 |
| 6 | **ceo-veto**（CEO 否决） | 最终资深裁决。可否决、下调信心或打回重来。它挑战的是*推理质量*，而非仅仅给出另一个比分。 | 关卡 3 |
| — | *编排器* | *并非子智能体。* 即 Claude Code 主会话，由 `CLAUDE.md` 管辖，通过 `/predict` 触发。 | — |

## 安装与使用

需要 [Claude Code](https://claude.com/claude-code) v2.x。

1. 克隆仓库：
   ```bash
   git clone https://github.com/tleung927/world_cup_2026_claude.git
   cd world_cup_2026_claude
   ```
2. 在项目目录中启动 Claude Code：
   ```bash
   claude
   ```
   `.claude/agents/` 与 `.claude/commands/` 会被自动识别为**项目级**子智能体和斜杠命令。
3. 确认智能体已加载 —— 在 Claude Code 中运行 `/agents`，应能看到全部 6 个。
4. 运行一次预测：
   ```
   /predict 法国 vs 瑞典，2026 世界杯 32 强赛，MetLife 球场
   ```
5. 最终预测会写入 `predictions/<日期>-<对阵>.md`。
6. 比赛结束后，记录结果：
   ```
   /postmatch 法国 2-1 瑞典 —— <实际发生了什么>
   ```

### 数据接入

智能体开箱即用地使用 `WebSearch` / `WebFetch`。首次运行时，Claude Code 会提示你允许各分析师抓取的域名（个人白名单会保存到被 git 忽略的 `.claude/settings.local.json`）。若日后接入付费数据源（Opta、SportRadar）或爬虫，把它做成 MCP 服务器或本地脚本，分析师智能体即可调用。

## 改进闭环

每场比赛结束后，`/postmatch` 会把结果追加到 `knowledge/lessons-learned.md`：预测了什么、实际结果如何、若判断错了 —— *属于哪一类*错误。「偏差审查」每次运行都会读取该文件，因此已记录的错误会主动防止自己重演。这正是整个系统的意义所在。
